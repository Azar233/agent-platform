# 数据集冻结校验规则

冻结前必须满足以下条件：

- `class_count` 大于 0，并与类别映射数量一致。
- `class_index` 从 0 开始连续排列，不能重复或跳号。
- `product_key` 在版本内唯一。
- 若使用 `product_id`，所有类别必须全部配置，不能只配置一部分；同一版本内不能重复。
- 普通训练数据集的 train、val、test 都必须有图片和标注。
- 必须存在 `content_hash`，用于标识冻结内容。

由可用模型导入的 `catalog_only` 版本可以没有训练图片和标注，但校验报告会给出警告。

启用文件系统检查时，本地 `storage_path` 必须是目录，`data.yaml` 必须存在，配置了 manifest 时对应文件也必须存在。远程 URI 会跳过本地文件检查并记录警告。校验报告中的错误会阻止冻结；警告不会自动阻止，但需要经营者确认其影响。
