# 数据集版本生命周期

VisionPay 的数据集版本状态包括 `draft`、`ready` 和 `archived`。

## 修改规则

- 历史冻结版本不能直接修改。
- 新增或删除商品样品前，应从已冻结版本派生一个草稿。
- 只有 `draft` 草稿可以添加样品、删除商品或修改版本内容。
- 冻结操作会先执行数据集校验；校验失败时不能冻结。
- 只有 `ready` 版本可以用于启动新版训练。
- 当前数据集通常不能直接归档，应先切换当前版本。

## 添加样品

添加样品有三种模式，用户必须明确选择，Agent 不得自行猜测：

1. 新建商品训练图：填写商品名称、类别英文名、价格和可选条码，只上传 train 图片。
2. 已有商品训练图：选择当前版本中的已有商品，只上传 train 图片。
3. val/test 结账场景：上传 val 和/或 test 场景图，每个检测框必须指定已有商品。

所有检测框都必须由用户在数据集页面逐图手工绘制并确认。Agent 只能解释规则、检查版本并发起页面交接，不能自动生成或确认检测框。

## 删除商品样品

当前业务按商品删除相关标注和必要图片，并重排后续 `class_index`。单商品图片会随商品删除；多商品场景仍有其他有效框时保留。删除前必须展示影响范围并要求二次确认。
