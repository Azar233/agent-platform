# 数据集删除操作的影响

## 删除商品样品

只允许在 `draft` 中执行。系统删除目标商品的全部标注；只包含该商品的图片会连同空标签文件一起删除，多商品场景仍有其他有效框时保留。被删除类别之后的 `class_index` 会整体减一，并重写 `data.yaml`、manifest 和标签文件。

操作前必须展示受影响图片数、删除标注数、保留的多商品场景数、重排类别数和是否同时停用稳定商品。`deactivate_product=true` 会把商品设为停用，但不会改变历史审计记录。

## 删除草稿

只允许删除非当前、没有派生子版本的 `draft`。系统永久删除该版本的图片索引、标注索引、类别映射、版本记录和托管文件。存在子版本时必须先处理派生关系，不能绕过校验。

## 归档不是删除

归档保留版本、训练和模型审计信息，但不再把版本作为可用训练版本展示。需要保留历史可追溯性时优先归档，不要用永久删除代替。
