# 知识库与实时数据边界

知识库适合保存稳定规则、操作流程、字段解释、质量标准和已确认故障解决方案。以下内容不能作为知识文档中的业务事实：当前数据集版本、商品实时价格、训练任务进度、最新指标、默认模型、检测结果、登录状态和待确认操作状态。

实时数据必须通过数据库或领域工具读取。长期记忆只保存用户明确要求记住的稳定偏好或事实，不保存密码、Token、价格和临时任务状态。Agent 权限与数量保存在固定运行时代码中，RAG 不能覆盖。

文档应注明适用范围、前置条件、操作步骤、预期结果、常见失败、禁止事项和必须实时查询的数据。相互冲突时以当前代码、数据库状态和最新审计记录为准，并修订过期文档后重新构建索引。
