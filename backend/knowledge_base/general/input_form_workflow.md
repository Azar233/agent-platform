# Agent 参数问询表单

当请求目标明确但缺少执行参数时，Agent 必须调用 `request_user_input_form`，不能用 Markdown 表格或自由文本临时设计字段。每个业务 `purpose` 的标题、字段、顺序、类型、默认值和提示由后端固定模板定义，因此同一操作的问询框应保持一致。

Agent 只能提供已知值和动态下拉选项，不能新增字段。表单提交包含 `form_id`；后端验证表单属于当前用户和会话、字段类型与必填规则，并拒绝重复提交。验证通过后请求强制返回最初发起表单的领域 Agent，不重新进行普通路由。

表单用于收集参数，不等于授权执行。只读查询可以在表单提交后立即调用工具；写操作仍要生成影响预览和独立确认卡。图片、文件夹选择、批量上传和检测框绘制不能塞进通用表单，应交给附件队列或专用页面。
