# 训练与模型监控

训练任务可以在本机、集群或离线环境执行。管理端通过任务记录、日志和指标进行监控，不应根据当前后端机器是否有 GPU 推断训练是否可用。

## 启动条件

- 新版训练入口应选择一个 `ready` 数据集版本。
- 启动前确认场景、数据集版本、基础模型、epoch、图像尺寸、batch、优化器和训练设备。
- 训练启动后保存任务 ID，通过状态接口查询进度，不保持单个 HTTP 请求长时间等待。

## 常用指标

- Precision：预测为商品的结果中有多少是正确的。低 Precision 常表现为误检较多。
- Recall：真实商品中有多少被检测出来。低 Recall 常表现为漏检较多。
- mAP：综合不同类别或 IoU 阈值下的检测精度。
- Loss：训练优化目标，应结合验证指标判断，不能只根据 Loss 下降发布模型。

## 发布约束

训练完成后应先评估和测试，再与当前默认模型比较。切换默认模型属于重要写操作，必须展示新旧模型、数据集来源和关键指标，并由经营者确认。
