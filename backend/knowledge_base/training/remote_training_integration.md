# 远程与离线训练集成

管理端负责登记任务、参数、状态、日志、指标和模型产物引用；实际训练可以在其他机器或集群完成。远程执行端应使用同一任务 UUID，并持续回传状态、当前 epoch、进度、错误、日志和逐 epoch 指标。

离线结果导入前核对数据集版本、类别顺序、模型结构、epochs、图像尺寸、batch、device 和最终状态。只允许导入 `completed`、`failed` 或 `cancelled` 的终态记录，不能把仍在运行的外部任务伪装成完成。

模型产物应包含可验证的权重路径和评估报告。导入完成的任务仍需验证模型并登记模型版本；没有活动模型时数据集保持 `pending_train`。网络中断时保留本地日志和结果文件，恢复后按任务 UUID 幂等同步，不能创建多个代表同一次训练的任务。
