# 训练参数选择指南

- `model_name` 支持 yolov11n/s/m/l/x。模型越大通常需要更多显存和训练时间，先用 n 或 s 验证数据与流程。
- `epochs` 是完整遍历训练集的轮数，默认 100。数据少时过多轮次可能过拟合，应结合验证指标和早停现象判断。
- `img_size` 默认 640，必须是合理的正整数；增大有利于小目标但显著增加显存和耗时。
- `batch_size` 默认 16。显存不足时优先减小 batch，再考虑降低图像尺寸。
- `device` 可填写 `cpu`、GPU 编号或运行环境支持的设备。请求 CUDA 但不可用时当前训练服务可能降级到 CPU，必须从日志确认实际设备。
- `optimizer` 支持 SGD、Adam、AdamW 和 auto；没有对比依据时使用项目默认值。
- `lr0` 是初始学习率，默认 0.01。异常震荡、Loss 不下降时再结合优化器调整，不要只凭单次曲线大幅修改。

参数必须通过训练表单和影响预览确认，不能从长期记忆自动沿用到高成本任务。
