# 误检与类别混淆排查决策树

1. 检查预测框是否落在相似包装、货架标签、海报或背景纹理上。
2. 适度提高 confidence，判断误检是否只是低置信度候选；同时检查正确目标是否因此漏掉。
3. 同一商品出现多个重叠框时检查 IoU/NMS 参数；相邻商品被合并或压掉时也需调整 IoU 对比。
4. 类别互相混淆时检查类别映射、相似商品样本数量、包装角度和人工标注一致性。
5. 为稳定背景误检补充不含目标的负样本，为相似商品补充能体现差异的近景和多环境样本。
6. 使用固定测试集比较修复前后 Precision、关键类别混淆和漏检，不能只看误检减少。

不要通过删除困难测试图片来制造更好指标，也不要用价格或商品名称规则覆盖模型错类。
